# 一、当前代码的主要问题

### 1️⃣ 单文件过大

现在这个模块基本：

```
builtins.rs ≈ 900+ 行
```

VBScript 内置函数类别：

```
日期函数
数学函数
字符串函数
数组函数
类型函数
随机函数
```

全部混在一个 `match` 里：

```
match name_lower.as_str() {
    "now" => ...
    "date" => ...
    "instr" => ...
    "split" => ...
    "join" => ...
}
```

后期会变成：

```
2000+ 行 match
```

维护难度很大。

------

### 2️⃣ 函数查找效率一般

现在每次调用：

```
name.to_lowercase()
match name_lower.as_str()
```

如果函数越来越多：

```
O(n) 匹配
```

虽然 n 不大，但结构不好。

------

### 3️⃣ 多参数 / 单参数函数分裂

你现在有：

```
call_builtin_function_multi()
call_builtin_function()
```

这是一个 **历史兼容设计**，其实可以统一。

------

### 4️⃣ 日期格式转换存在 bug

这里有个 **隐藏问题**：

```
.replace("MM", "%m")
.replace("DD", "%d")
.replace("HH", "%H")
.replace("MM", "%M")
```

`MM` 被替换两次：

```
月份 -> %m
分钟 -> %M
```

会冲突。

# 三、统一函数调用接口

建议统一为：

```
pub fn call_builtin(
    name: &str,
    args: &[Value]
) -> Option<Result<Value, RuntimeError>>
```

然后 dispatch：

```
match name {
    "instr" => string::instr(args),
    "split" => string::split(args),
    "abs" => math::abs(args),
    "now" => date::now(args),
    _ => None
}
```

Interpreter 调用：

```
if let Some(result) = builtins::call_builtin(name, &args) {
    return result;
}
```

这样就不用：

```
call_builtin_function_multi
call_builtin_function
```

# 四、进一步优化（推荐）

如果你想 **更优雅一点**，可以用 **函数注册表**。

例如：

```
type BuiltinFn = fn(&[Value]) -> Result<Value, RuntimeError>;
```

注册：

```
static BUILTINS: phf::Map<&'static str, BuiltinFn> = phf_map! {
    "instr" => string::instr,
    "split" => string::split,
    "abs" => math::abs,
    "now" => date::now,
};
```

调用：

```
if let Some(func) = BUILTINS.get(name) {
    return Some(func(args));
}
```

优点：

```
O(1) 查找
结构非常清晰
```

# 五、字符串函数可以复用 helper

例如你有很多：

```
ValueConversion::to_string(&args[0])
```

建议写 helper：

```
fn arg_string(args: &[Value], i: usize) -> String
fn arg_number(args: &[Value], i: usize) -> f64
fn arg_bool(args: &[Value], i: usize) -> bool
```

这样代码会干净很多：

现在：

```
let string = ValueConversion::to_string(&args[0]);
```

优化后：

```
let string = arg_string(args, 0);
```

# 六、日期函数建议改进

VBScript 日期类型其实是：

```
Double
```

表示：

```
days since 1899-12-30
```

现在你全部返回：

```
Value::String
```

这会导致：

```
DateAdd + DateDiff 计算困难
```

更好的设计：

```
Value::Date(f64)
```

但这是 **解释器升级版本**，不是必须。

# 九、一个非常重要的优化

你现在很多：

```
ValueConversion::to_string
ValueConversion::to_number
```

建议把它们变成 trait：

```
trait VbValueExt {
    fn to_str(&self) -> String;
    fn to_num(&self) -> f64;
    fn to_bool(&self) -> bool;
}
```

然后：

```
arg.to_num()
arg.to_str()
```

解释器代码会 **短很多**。
